# Odaeri 포트폴리오

## 1. 프로젝트 개요
Odaeri는 음식 주문과 픽업을 지원하는 iOS 앱으로, 고객용(User)과 관리자용(Admin) 두 개의 타겟으로 구성되어 있습니다.

- 고객용(Odaeri_User): iPhone 전용, iOS 16.1+, Clean Architecture + MVVM-C
- 관리자용(Odaeri_Admin): iPad 전용, iOS 16.1+, Simple MVVM
- 공통 코드: 네트워크, 모델, 유틸리티, 디자인 시스템을 Shared에서 공유

## 2. 공통 아키텍처/기술 스택

### 아키텍처
- User: Presentation -> Domain -> Data 계층 분리
- Admin: MVVM 기반 단순 구조로 빠른 CRUD 대응
- Coordinator 패턴으로 화면 전환 로직 분리

### 네트워크
- Moya/Alamofire 기반 API 계층화
- 토큰 갱신 동시성 제어(TokenRefreshCoordinator)로 중복 호출 방지
- API/DTO/Entity 분리로 안정적인 모델 변환

### 로컬 저장
- Realm 기반 로컬 캐시
- 유저별 Realm 분리 및 암호화 키(Keychain 연동) 설계
- 스키마/마이그레이션 대응을 위한 구성 분리

### 디자인 시스템
- AppColor/AppFont/AppImage 기반 스타일 통일
- 공통 컴포넌트 재사용성 강화

# Odaeri_User (고객 앱)

## 1. 핵심 기능

### 1) 인증
- Apple/Kakao 소셜 로그인 연동
- Access/Refresh Token 관리 및 자동 재발급
- 토큰 만료/중복 갱신 레이스 조건 방지

### 2) 커뮤니티
- 게시글 리스트/상세/댓글 플로우
- 미디어 그리드 및 상세 뷰 구성
- 데이터 바인딩 및 상태 기반 UI 갱신

### 3) 채팅
- Socket + REST 하이브리드 실시간 채팅
- Realm 로컬 저장, 역방향 페이지네이션
- 전송 실패/재시도/임시 메시지 상태 처리
- 이미지/파일/비디오 첨부 및 카드형 공유 메시지 대응

### 4) 주문/결제
- 주문 생성 -> PG 결제 -> 영수증 검증 -> 주문 확정 플로우
- 중복 결제/네트워크 단절 대비 로직 고려
- 영수증 검증 실패 시 예외 처리 경로 설계

### 5) 스트리밍 (HLS)
- 스트리밍 URL 요청 후 AVPlayer 재생
- 품질별 HLS 플레이리스트 지원
- 자막 파일 다운로드 및 재생 시간 동기화
- Stream URL 캐싱 및 이미지 캐싱 최적화

### 6) 길찾기/내비게이션
- 경로 안내 및 상태 관리
- TTS 안내 및 위치 추적 관리 구조 마련

### 7) 배너/웹뷰
- 웹뷰 브릿징을 통한 이벤트/데이터 연동

### 8) Live Activity
- 픽업 상태를 실시간으로 위젯에 표시

## 2. 상세 기술 정리

### A. 채팅

#### 구현 내용
- Socket + REST 하이브리드 구조
- Realm 기반 로컬 저장 및 역방향 페이지네이션
- 이미지/파일/비디오 첨부 처리
- 카드형 공유 메시지 분기 처리

#### 고려 사항
- 동시성 충돌 방지(동일 메시지 중복 저장 방지)
- 전송 실패/재시도 상태 표시
- 대용량 메시지 로딩 시 스크롤 성능 유지

#### 성능 최적화
- 메시지 매핑/정렬은 백그라운드에서 수행
- DateFormatter 및 문자열 계산 캐싱
- 뷰 Lazy 생성 및 재사용 최적화

---

### B. 스트리밍

#### 구현 내용
- HLS 마스터 플레이리스트 기반 재생
- 품질별 URL 선택 및 캐싱
- 자막 파일 다운로드 및 파싱 후 시간 동기화

#### 고려 사항
- 토큰 포함된 스트림 URL 재사용
- 네트워크 상태 변화 대응
- 자막 자동 스크롤 및 수동 스크롤 전환

#### 성능 최적화
- Stream URL 캐싱
- 이미지 메모리/디스크 캐시 활용

---

### C. 결제

#### 구현 내용
- 주문 생성 -> 결제 -> 영수증 검증 -> 확정
- PaymentEntity 기반 상태 관리

#### 고려 사항
- 네트워크 단절 시 영수증 재검증 흐름
- 중복 결제 방지(결제 상태 기준 검증)
- 실패 시 사용자 알림과 재시도 분기

---

### D. 보안/세션

#### 구현 내용
- Refresh Token 관리 및 자동 갱신
- TokenRefreshCoordinator 기반 동시성 제어

#### 고려 사항
- 중복 요청 방지
- 갱신 실패 시 안전한 로그아웃 처리

# Odaeri_Admin (관리자 앱)

## 1. 핵심 기능

### 1) 가게/메뉴 등록 및 관리
- Store 등록/수정/이미지 업로드 지원
- Menu 등록/수정/이미지 업로드 지원
- Admin 전용 API(StoreAPI+Admin, MenuAPI) 연동

### 2) 주문 관리
- 처리 중/완료 상태 분리
- 상태 변경(updateOrderStatus) 연동
- 주문 리스트 + 상세 분리 UI
- 상태 기반 뷰 전환

### 3) 매출 조회
- 기간 필터(시작~종료)
- KPI 요약(매출액/주문수/객단가/평균별점)
- 시간대별 매출 추이(막대)
- 카테고리별 판매 비중(도넛)
- 인기 메뉴 TOP5

### 4) 설정
- 로그아웃 및 관리 설정 진입

---

## 2. 상세 기술 정리

### A. 가게/메뉴 관리

#### 구현 내용
- StoreRequest/MenuRequest 기반 등록/수정
- 이미지 업로드 및 미디어 API 연동

#### 고려 사항
- 이미지 교체/미리보기 UX
- 등록/수정 상태 분기 처리

---

### B. 주문 관리

#### 구현 내용
- 주문 상태별 리스트 분리
- 상세 화면에서 고객/메뉴/픽업 정보 분리

#### 고려 사항
- 운영 우선순위가 높은 주문을 빠르게 식별
- 진행/완료 상태 전환 시 즉시 반영

---

### C. 매출 조회

#### 구현 내용
- 기간 필터(시작~종료) 기반 데이터 필터링
- KPI 카드 + 차트 조합
- Swift Charts 기반 시각화

#### 고려 사항
- 기간별 대량 데이터 처리 성능
- 시각적 인사이트 강조(피크 타임, 인기 메뉴)

- KPI + 차트 기반 매출 분석
- 기간 필터 및 실시간 인사이트 제공
